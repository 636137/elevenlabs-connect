ISSUE: Widget plays agent audio, but user microphone input is not sent.

ROOT CAUSE: Usually one of these:
1. Microphone permission not granted in browser
2. Agent ID not configured (still says AGENT_ID_HERE)
3. Not using HTTPS or localhost
4. Browser extension blocking microphone
5. Incorrect agent configuration

  - Copy your agent ID

STEP 2: Test with diagnostics page
  - Open: elevenlabs-connect/test_audio_connection.html
  - Paste Agent ID

  - Grant microphone permission when prompted

STEP 3: Fix any issues
  - Read AUDIO_TROUBLESHOOTING.md for your specific error

STEP 4: Use the widget
  - Open widget_demo.html
  - Replace AGENT_ID_HERE with your actual ID
  - Click widget button (bottom-right)
  - Start conversation
  - Speak clearly after agent finishes
